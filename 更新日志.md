# 更新日志 (Changelog)

## v0.0.6 (2026-04-17)

### 🚀 新特性与优化
- **核心指纹伪装 (Impersonator)**: 引入 `reqwest-impersonate` 库，实现了对 **Chrome 124** 级别的 TLS/HTTP2 指纹模拟，大幅提升了对 Cloudflare 和 OpenAI 防火墙的穿透能力。
- **浏览器行为对齐**: 
    - 统一了全局 `User-Agent` 为 Chrome 124 版本。
    - 自动化注入 **Client Hints** (`sec-ch-ua`, `sec-ch-ua-platform` 等)，确保协议层与应用层特征完美契合。
    - 默认注入 `en-US` 语言头部，优化全球代理环境下的隐身效果。
- **账号状态检查引擎升级**:
    - 现在能精确区分 **401 (Expired/过期)** 与 **403 (Banned/封禁)** 状态。
    - 检查逻辑接入指纹伪装，避免因检查行为本身触发风控。
- **浏览器自动化注册增强**:
    - 升级了 Token 扫描脚本。
    - 新增对 `/backend-api/session` 接口的探测。
    - 引入对 `window.__NEXT_DATA__` 全局对象的深层递归扫描，即便在复杂环境下也能通过内存提取遗落的 **Access Token (API)** 与 **Refresh Token**。
    - 优化了 `fetch` 调用的凭据策略 (`credentials: 'same-origin'`)，确保鉴权接口稳定走通。

### 🛠️ 修复与改进
- **前端设置页面**:
    - 修复了 `SettingsView` 中 `Database` 图标找不到的错误。
    - 修复了 `SettingsPayload` 类型定义中缺失 `sub2api_url` 和 `sub2api_key` 的问题。
    - 修正了 CPA 授权流程中对返回数据 (`res.data`) 的类型访问错误。
- **后端配置系统**:
    - 补全了后端 `SettingsPayload` 结构体，支持持久化存储 Sub2API/NewAPI 的分发配置。
- **代码整洁度**: 
    - 重构了 `register.rs` 和 `sentinel.rs`，移除零散且不一致的硬编码头部，采用集中化的模拟器提供者模式。
