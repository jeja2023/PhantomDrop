## [V0.0.18] - 2026-05-10

### 🛡️ 管理面认证与密钥暴露加固
- **移除 URL Token 认证**：后端认证中间件不再接受 `?token=` 查询参数，避免认证密钥进入浏览器历史、代理日志和 Referer。
- **管理面默认锁定**：`/api/*`、`/stream`、`/console` 在未配置 `auth_secret` 或 `HUB_SECRET` 时返回 `503` 并拒绝访问，避免空密钥状态下管理接口被误放行。
- **认证来源收敛**：管理面认证统一支持 `Authorization: Bearer`、`X-Auth-Token` 与 `phantom_auth_token` Cookie；`/health` 与 `/ingest` 保持独立接入路径。
- **前端 Cookie/SSE 适配**：Web 登录后同步写入 `phantom_auth_token` Cookie，SSE 改为 `EventSource(..., { withCredentials: true })`，不再把令牌拼接到 `/stream` URL。

### 🔐 日志脱敏与敏感信息保护
- **工作流日志红action**：写入数据库和推送 SSE 前统一脱敏密码、密令、token/key/secret 字段、JWT 片段以及 `sess_` 会话令牌。
- **错误输出收敛**：减少敏感凭据在 UI 日志、工作流步骤和系统事件中的泄漏风险，保留必要的排障上下文。

### 🧭 账号中心与前端质量修复
- **账号中心重建修复**：修复 `AccountListView` 历史编码损坏导致的 JSX 解析错误，恢复账号搜索、分页、跨页全选、批量状态检查、CPA 上传、批量删除、CSV/JSON 导出和 Token 详情弹窗。
- **类型与 Hook 清理**：补齐账号中心、注册中心和自动化视图中的响应类型、错误处理与 Hook 依赖，消除 ESLint 错误和 stale closure 风险。
- **Vite 配置整理**：移除已失效的 Tailwind 类型抑制注释，保持 TypeScript 构建链干净。

## [V0.0.17] - 2026-05-10

### 🤖 OpenAI 浏览器注册流程修复
- **注册入口兼容新版中转页**：增强 `openai_browser_register` 的 ChatGPT 注册入口识别能力，支持自动点击新版 `Get started` 页面中的 `Sign up for free` / `Sign up` / `免费注册` / `注册` 入口。
- **注册路径兜底切换**：当页面持续停留在 `auth/login?screen_hint=signup` 时，会自动尝试切换到 `https://chatgpt.com/auth/signup`，减少因 OpenAI 前端入口调整导致的 `email_not_found`。
- **备用入口增强**：浏览器注册会依次尝试 `chatgpt.com/signup`、`chatgpt.com/`、`auth.openai.com/u/signup` 与 `auth0.openai.com/u/signup` 等入口，降低单一路径失效导致的注册失败率。
- **失败诊断增强**：当邮箱输入框缺失时，日志会输出当前页面 URL、标题、可见按钮、输入框与正文摘要，即使生产环境关闭 `/debug` 快照也能定位卡点。
- **验证码优先流程兼容**：浏览器注册状态机新增“先邮箱验证码、后设置密码”的路径，兼容当前 ChatGPT 注册页先显示 `Check your inbox` 再继续设置密码的新顺序。
- **无密码注册自适应流支持**：针对 OpenAI 最新大范围下线的“无密码注册流”（即输入邮箱 $\rightarrow$ 邮件 OTP 验证码 $\rightarrow$ 填写姓名和年龄后即直接完成注册，无密码设置），在浏览器驱动中实现了智能双向竞争嗅探。可同时检测密码设置框或资料页，遇到无密流程时自动标识 `Passwordless Flow` 并智能跳过密码填充，保障全流程跑通。
- **中文邮箱验证页面精准匹配**：在原有支持英文 `'Check your inbox'` 页面特征的基础上，新增了中文环境下官方 `'检查你的收件箱'` 标志词的适配，大幅提升了中文浏览器或代理网络下的页面特征判识准确率。
- **调查问卷与新手弹窗智能跳过**：针对新版 OpenAI 注册完成后弹出的“是什么促使你使用 ChatGPT？”等问卷与引导流程，扩展了后续确认拦截库，自动定位并模拟点击 `'Skip'` 与中文 `'跳过'` 按钮，帮助新产出的账号瞬间进入 Dashboard 抓取 Token。

### ✉️ OTP 解析与邮件轮询修复
- **OpenAI 专用验证码提取**：OpenAI 发件人的邮件入库时优先使用 6 位数字 OTP 专用解析，避免通用解析规则误伤官方 HTML 模板。
- **误提取防护**：修复将 `Enter` 等纯英文提示词误识别为验证码的问题，通用字母数字验证码现在必须至少包含一个数字。
- **轮询结果收紧**：内部 OTP 轮询接口只返回 6 位数字验证码，历史误入库的 `Enter` 等非验证码内容不会再被注册流程消费。
- **OpenAI 回弹域名识别**：补充识别 `em****.tm.openai.com` 等 OpenAI 邮件回弹发件域，确保 `bounces+...@em7877.tm.openai.com` 这类验证码邮件也会走专用 OTP 解析。
- **分散数字验证码兼容**：支持从 HTML 模板清洗后的 `1 2 3 4 5 6` 这类分散数字中合并提取 6 位验证码。
- **回归测试补充**：新增解析器测试，覆盖 `verification code Enter ...` 场景，确保不会再把按钮或提示文案当作验证码。

### 📬 邮件详情完整正文展示
- **HTML 正文兜底展示**：邮件详情弹窗不再只展示 `body_text`；当纯文本为空时，会自动将 `body_html` 清洗为可读正文展示。
- **原始 HTML 排查视图**：新增 `RAW HTML` 区块，可直接查看和复制原始 HTML，方便定位验证码、链接或模板隐藏内容。
- **复制体验修正**：正文复制按钮现在复制实际展示内容；HTML 区块提供独立复制按钮，提升排查邮件解析问题时的效率。
- **弹窗布局修复**：邮件详情弹窗改为通过 React Portal 挂载到 `document.body` 的全屏居中 overlay，避免页面侧边栏、顶部栏或动画容器影响 `fixed` 定位，并收紧头部高度，确保标题、Trace ID 与关闭按钮完整显示。

### 🛰️ Cloudflare 邮件中继修复
- **Worker 转发失败不再静默吞错**：当 Hub `/ingest` 返回 401/5xx 或网络错误时，Email Worker 会抛出异常并记录 Hub 状态、收件人、发件人和主题，避免 Cloudflare 显示已送达但 Hub 实际未入库时无从排查。
- **Worker Secret 自动同步**：Cloudflare 自动化脚本部署 Worker 后会将 Web 全局设置中的接口令牌同步到 Cloudflare Worker Secret Store 的 `HUB_SECRET`，避免 Docker 重建后 Worker 仍使用旧令牌导致邮件被 Hub 拒收。
- **接口令牌优先级修正**：自动化脚本优先使用 Web 全局设置 `auth_secret`，再使用自动化缓存，最后才使用环境变量 `HUB_SECRET` 兜底，避免服务器环境变量覆盖用户在控制台修改的新令牌。
- **部署脚本稳健性修复**：`wrangler.toml` 不再写入明文 `HUB_SECRET`，并修复部署脚本在部分 PowerShell 编码环境下的解析风险。
- **Hub 入库诊断增强**：后端 `/ingest` 成功或失败都会输出邮件 ID、发件人、收件人、主题和验证码提取结果，便于直接通过 Docker 日志判断邮件是否真正进入数据库。

## [V0.0.16] - 2026-05-10

### 🧱 后端架构与路由模块化
- **主入口瘦身**：将原本集中在 `core/src/main.rs` 的大段路由定义拆出到 `core/src/routes.rs`，`main.rs` 现在主要负责配置加载、数据库初始化、SSE、工作流、隧道与监听启动。
- **邮件路由子模块**：新增 `core/src/routes/emails.rs`，集中管理邮件列表、详情、归档、批量删除以及 `/ingest` 注入入口，降低后续继续拆分账号、工作流、设置路由的成本。
- **Axum 状态类型对齐**：邮件子路由保持 `Router<Arc<StreamHub>>`，与根路由 `with_state(stream_hub)` 保持一致，避免拆分后 SSE 状态推断冲突。

### 🗄️ 数据库迁移与 SQLite 稳定性
- **迁移体系正规化**：新增 `core/migrations/202605100001_initial_schema.sql`，将核心表结构与索引迁入 `sqlx::migrate!()` 管理。
- **旧库兼容补列**：保留旧数据库升级能力，通过 `PRAGMA table_info` 检查缺失字段后再按需 `ALTER TABLE`，替代过去盲目静默执行多条 `ALTER TABLE` 的方式。
- **SQLite 运行参数增强**：启动时配置 WAL、`synchronous=NORMAL`、`busy_timeout=5000` 与外键开关，提升持续写入和控制台查询时的稳定性。
- **关键查询索引补强**：新增邮件时间、归档邮件时间、账号创建时间等索引，优化邮件列表、归档列表和账号中心查询路径。

### 🛡️ 生产安全与运行配置
- **集中化运行配置**：新增 `core/src/config.rs`，统一管理 `APP_ENV`、`BIND_ADDR`、`PORT`、`CORS_ORIGINS`、调试资源开关与生产密钥校验。
- **生产密钥防呆**：生产环境使用默认 `local_dev_secret` 或未配置 `HUB_SECRET` 时，后端会拒绝启动。
- **CORS 收紧**：后端不再使用完全开放的 permissive CORS，改为按环境变量配置允许来源、方法和请求头。
- **调试截图接口加固**：`/debug/:name` 仅允许安全的 `.png` 文件名，生产环境默认关闭，避免路径穿越和调试资产误暴露。
- **OAuth 凭据输出收敛**：CPA OAuth 交换接口不再向前端返回完整认证数据，仅返回状态与邮箱；写出 `codex_auth.json` 改为 `WRITE_CODEX_AUTH_FILE=true` 时才启用。

### ✉️ 邮件解析与后端测试修复
- **Rust Regex 兼容修复**：修复 OpenAI OTP 正则中 Rust `regex` 不支持 look-around 导致的测试失败问题。
- **解析边界测试补充**：新增独立 6 位验证码提取测试，避免从长数字中误截 OTP。
