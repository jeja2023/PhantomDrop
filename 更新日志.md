# 更新日志 (CHANGELOG)

## [V0.0.10] - 2026-04-13

### 🤖 OpenAI 浏览器仿真引擎优化 (PhantomBrowser Overhaul)
- **无头模式动态切换 (Headless Toggle)**：在“通用注册中心”界面新增了“无头模式”开关。支持在本地开发环境开启“有头”窗口实时观察，或在云服务器环境下开启“无头”静默运行。
- **引擎名称汉化**：将注册引擎选项翻译为更直观的“协议模式”与“浏览器模式”，优化了中文用户的操作体验。
- **自动截图调试机制**：为无头模式引入了“快照回传”功能。在遇到 Cloudflare 拦截或元素超时时，系统会自动生成 `debug_xxx.png` 截图并存证，解决了云服务器环境下“看不见、难调试”的痛点。

### 🛡️ 隐身能力与检测绕过 (Anti-Detection)
- **WebDriver 特征抹除**：通过底层 CDP 指令注入，动态清除了 `navigator.webdriver` 标识。
- **高权重 UA 伪装**：自定义了全套浏览器启动参数，包括高权重 User-Agent 伪装和内存溢出防护 (`--disable-dev-shm-usage`)，显著提升了通过 OpenAI 风控网关的概率。

### 🐳 Docker 运行时环境补全
- **浏览器环境预装**：大幅更新了 `Dockerfile`，在镜像内完整集成了 `chromium` 浏览器及所有运行依赖库。
- **多语言字体支持**：预装了常用的中文字体库（文泉驿等），彻底解决了浏览器在后台截图中出现的中文乱码和方块字问题。

### 🛠️ 稳定性与 API 适配修复
- **接口兼容性固化**：修复了 `headless_chrome` 1.0.x 版本在不同环境下的 API 差异问题，确保了截图枚举值和脚本注入逻辑在全平台的一致性。

### 🎨 注册中心 UI/UX 深度改进
- **无感刷新架构**：重构了任务队列和日志流的轮询逻辑，引入深度对比更新机制，彻底解决了页面在自动刷新时出现的“一闪一闪”现象。

- **流程自动重定向**：点击“启动”后将自动清屏旧日志并锁定到新生成的任务，操作体验更加连贯。

### 🛠️ 稳定性与类型系统修复
- **TypeScript 严格化**：修复了 `RegistrationView` 中 `postJson` 泛型参数缺失导致的编译错误，提升了前端代码的健壮性。
- **Sentinel 错误分析**：针对 500 错误增加了详细的异常捕获与解析逻辑，方便在复杂网络环境下快速排查 IP 封禁问题。

## [V0.0.7] - 2026-04-11

### 🤖 OpenAI 原生自动化注册引擎 (Native OpenAI Engine)
- **协议级注册链路集成**：彻底告别模拟逻辑。实现了从 PoW 解算 (Sentinel)、PKCE 参数构造、OAuth 授权到 OTP 二次验证的全链路协议级对接。
- **高性能算力解算 (PoW Solver)**：集成了高效的 Rust 原生 PoW 解算器，支持实时处理 OpenAI 的算力挑战，显著提升了自动化注册的成功率和稳定性。
- **双阶段注册状态机**：重构了注册流为两阶段状态机（Phase A: 账号激活，Phase B: Token 捕获）。支持自动截取并持久化 Access Token、Refresh Token、Session Token 以及 Device ID。
- **智能 OTP 语义轮询**：优化了针对 OpenAI 邮件的正则表达式解析器，并建立了毫秒级的数据库轮询探针，确保验证码在邮件落盘后即刻被捕获并回填注册流。

### 🏢 通用注册中心与产物治理 (Universal Registration Center)
- **全新注册视图 (RegistrationView)**：Web 界面新增“通用注册中心”。支持实时观察注册任务的每一步原子逻辑，并提供注册产物的可视化明细列表。
- **账号产物全生命周期管理**：数据库 schema 升级，支持完整记录账号的 Token 序列、工作区标识及分发状态。实现了 `generated_accounts` 表的深度索引。
- **多平台分发协议 (CPA Uploader)**：内置账号自动分发模块。注册成功的产物可根据配置自动推送至第三方 CPA 平台或分发中枢。

### 📡 实时运行态监控增强
- **高频 SSE 步骤流**：为 `openai_register` 工作流专门定制了高频 SSE 推送策略。前端 `LogsView` 和 `RegistrationView` 现在可以实时渲染注册引擎的每一次内部网络请求状态。
- **任务产物快照**：支持在工作流运行过程中实时更新产物汇总。可在不中断当前任务的情况下，直接从界面导出已完成的账号 CSV 或 JSON。

## [V0.0.6] - 2026-04-11

### 🚀 开发体验与环境治理
- **开发端口同步**：修复了本地开发环境与生产环境接口不一致导致的 502 Bad Gateway 问题。将 Vite 代理、Powershell 脚本探针以及 Rust 后端核心的启动端口统一调整为 `9010`，确保一键启动流程的顺畅。
- **命令行乱码修正**：为 `启动开发环境.ps1` 增加了显式的 UTF-8 编码声明 (`[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`)，彻底解决了在部分默认编码非 UTF-8 的 Windows 终端中运行脚本时出现的中文乱码报错问题。

### 🚀 自动化脚本鲁棒性增强
- **部署重试机制**：为 `setup-cloudflare-mail.ps1` 脚本引入了健康检查重试循环（5次尝试/5秒间隔）。解决了 Cloudflare Worker 部署后因全球网络生效延迟（DNS/路由传播）导致的 404 误报问题。
- **全效路由自动化**：默认开启 Email Catch-all 转发功能。现在脚本在确保特定地址可用的同时，会自动配置域名全员抓取，极大提升了多账号注册的便利性。

### 🧠 后端生成逻辑重构 (Account Generation Overhaul)
- **随机账号算法优化**：彻底重构了 `core` 后端的账号生成逻辑。移除了 `pd_` 前缀和时间戳标识，改为生成 **8-12 位** 纯随机、无规律的数字和小写字母组合，提升了在目标网站注册时的拟真度。
- **高熵随机性增强**：引入了 Rust 原生 `rand` 库替代简单的 UUID 截取，确保生成的邮箱前缀具有更高的熵值和不可预测性。

### 🎨 前端交互体验深度优化
- **邮件详情视图优化**：修复了邮件详情页弹窗在低分辨率/长内容下导致“标题栏被切断”和重叠的排版问题。应用了严谨的 `flex-shrink: 0` 与居中对齐逻辑，并对全屏模态框外层间距、内部网格信息卡片的对齐方式、长文本的高度做了全局瘦身控制，使弹窗更加紧凑顺眼。
- **步骤明细“无感刷新”**：重构了 `AutomationView` 的状态管理。实现了步骤轨迹的后台静默更新，解决了之前轮询刷新时整个 UI 区域频繁闪烁、高度跳变和 Content Layout Shift 问题。现在只有在初次加载时才会看到 Loading 动画。

## [V0.0.5] - 2026-04-09

### 🏗️ 架构演进：单镜像交付 (Single Image Delivery)
- **单镜像封装**：实现前后端同镜像打包。基于多阶段构建，将 Rust 后端与 React 前端整合成一个标准的 Docker 镜像，部署门槛降至最低。
- **Axum 静态托管**：后端核心新增 `tower-http` 模块，支持原生托管前端 SPA 产物。现在的后端不仅仅是 API，也是全功能的 Web 服务器。
- **自动化运行时增强**：Docker 镜像现在完整内置了 `pwsh` (PowerShell Core)、`Node.js`、`npm` 和 `cloudflared`。解决了之前容器环境下无法运行 Cloudflare 自动化脚本的痛点。
- **智能路径探测**：后端逻辑升级，支持通过 `WEB_DIST` 环境变量灵活指定前端产物路径，并自动兼容开发与容器运行环境。
- **诊断入口搬迁**：原有的内建诊断控制台（Console）从 `/` 迁移至 `/diag`，为正式 Web 界面腾出主域名空间。

### 🧹 环境治理与生产就绪
- **统一配置模板**：根目录新增 `.env.example`。标准化了 `HUB_SECRET`、`PHANTOM_DB_URL` 及 Cloudflare 令牌等核心配置项的管理。
- **深度目录清理**：移除了冗余的 `docker/` 子目录及其陈旧的 Nginx 配置文件。现在根目录下的 `Dockerfile` 和 `docker-compose.yml` 是唯一的真理。
- **清理工程噪音**：自动清理了本地残留的 `dev-*.log` 日志、`web/dist` 构建缓存以及旧的备份目录 `.git.local-backup`，确保 Git 仓库和镜像构建的纯净度。
- **持久化策略固化**：在 `docker-compose` 中明确了 `./data` 和 `./.automation` 的挂载要求，确保数据库和自动化任务状态在容器销毁后依然存在。

## [V0.0.4] - 2026-04-08

### 🌐 Web 主界面对接与收尾
- `web` 开发环境新增 Vite 代理，`/api`、`/stream`、`/health` 会自动转发到 Rust 后端，开发时不再误拿到 Vite 的 HTML 回退页。
- 前端 API 层新增非 JSON 响应保护，接口地址或代理配置错误时会直接抛出可读错误，而不是出现 `Unexpected token '<'`。
- `web` 已切换为真实后端协议：工作流运行记录改为消费分页对象返回，命令面板不再拼接旧式中文工作流 ID，而是按 `kind` 解析真实工作流定义后再触发。
- 自动化工作流页接入服务端分页、状态筛选、工作流 ID 精确/模糊匹配和每页数量切换，不再只固定读取第一页最近记录。
- 全局数据流补齐：主界面启动时会同步邮件、统计、工作流运行记录、最新步骤流与设置值，邮件归档变更也会回刷统计。
- `web` 主界面新增全局错误边界，页面运行时崩溃时会落到可恢复兜底页，并可直接跳转到 Rust 内建控制台。
- 顶部导航栏补齐后端健康状态与实时流状态显示，支持在不中断当前页面的情况下观察 API 与 SSE 链路状态。
- 修复邮件验证码提取误命中，`verification code is 123456` 这类文本不再错误提取为字符串 `code`。
- 修复邮件分页接口的归档筛选链路，单封邮件归档后可通过 `archived=true` 正常查询到归档结果。

### 🎨 主界面体验统一
- `Dashboard`、`Emails`、`Logs`、`Tunnel`、`Automation`、`Settings` 六个主页面统一切换到 `page-shell`、`page-shell--full`、`PageHeader` 和 `page-panel` 这套布局骨架。
- 页面标题行、左右留白、主内容最大宽度、主面板圆角/边框/阴影、数据面板头部节奏已收敛为统一视觉语言。
- “系统流实时监控”页重构为 `原始加密流审计`、`运行态摘要`、`工作流步骤流` 三栏布局，右侧监控信息层级更清晰。
- “实时控制中心”页右侧“决策核心”面板压缩了信息密度与统计布局，保证“进入高级模式”按钮可见。
- “内网穿透助手”页去除了重复说明带，状态卡、配置区、链路拓扑区整体压缩，并将 `TUNNEL_SYNC` 三个输入项收敛到同一行显示。
- “系统全局设置”页压缩了上下内容节奏，输入控件更紧凑，保存按钮改为粘底操作条，保证低高度窗口下依然可操作。
- “自动化工作流”页工作流卡片、最近执行记录、执行步骤明细与任务产物均做了密度优化，支持在同屏展示更多记录。

- 全局顶部导航栏恢复为统一高度与间距，避免 `Automation`、`Settings` 等页面切换时出现“系统就绪”区域被压缩或节奏漂移。
- “系统流实时监控”页右侧两栏重排为真正的中栏 `运行态摘要` 与右栏 `工作流步骤流`，并进一步压缩摘要卡高度，为步骤流让出可视空间。
- “系统全局设置”页已切换到与其它主页面一致的整屏容器约束和统一页头系统，不再单独使用旧式页面结构。
- `web` 主界面新增统一的按钮与表单尺寸 token，主要按钮、分页按钮、导出按钮、危险按钮、筛选下拉与输入框开始收敛到同一套 `phantom-btn` / `phantom-input` / `phantom-select` 规格。
- 全局数据表尺寸同步统一：表头、单元格内边距与字号已在 `phantom-table` 上收敛，邮件列表等主数据表页面的按钮尺寸也已同步对齐。
- `自动化工作流` 页进一步压缩：工作流卡片更扁平，最近执行记录、执行步骤明细与任务产物已支持更紧凑的双列信息布局，便于同屏展示更多内容。
- 新增 `setup-cloudflare-mail.ps1`，将 `trycloudflare`、公网 IP、公网域名三种 Worker 对接路径统一成一套自动化脚本。
- 新增 `initialize-cloudflare-automation.ps1`，用于首次授权后固化 Cloudflare 凭据、Zone 信息、默认模式与默认公网地址，使后续对接可以零参数运行。
- `network` Worker 新增 `/health` 与 `/relay-test` 诊断入口，便于在不依赖真实邮件投递的前提下自动验证 `Worker -> /ingest` 链路。
- 新增 `Cloudflare接入自动化说明.md`，固定三种公网接入方案的逻辑流程、依赖条件与测试顺序。

### 🧰 开发与运维入口
- 新增 `启动开发环境.ps1`，默认同时启动 `core` 与 `web`，并自动打开 `http://127.0.0.1:5173/`。

- Rust 原生控制台保留为最小维护的兜底入口，日常主使用路径已切换为 `web`。

### ✨ 核心功能发布 (Core Features)
- **🎉 极客美学面板 (Phantom UI)**: 发布了基于 Dark Mode Glassmorphism 设计的全新 Dashboard，支持动态点阵背景和毛玻璃模糊滤镜。
- **⚡ 魔法命令中枢 (Cmd+K)**: 实现了全局命令弹出层，支持快捷键呼出并直接输入指令，提升重度用户操作效率。
- **🎮 Canvas 网格渲染引擎 (WebGL)**: 引入了基于 Canvas 2D 的高性能渲染网格，确保海量邮件数据流达到满帧 60fps 的实时流畅滚动。
- **🧠 智能解析器 (Neural Parser)**: 后端集成 regex 预热机制的智能提取引擎，自动从邮件文本中捕获 4-8 位验证码。
- **📡 SSE 实时流枢纽 (Stream Hub)**: 实现了全双工 SSE 实时推送链路，抛弃轮询，确保边缘节点捕获的数据秒级上屏。
- **🛡️ 边缘拦截节点 (Email Worker)**: 发布了适配 Cloudflare Workers 的边缘拦截应用，支持隔离 MIME 解码与加密推送。

### 🛠️ 全栈架构优化 (Refactoring)
- **📁 目录精简化**: 按照轻量化规范重命名字根目录为 `core` (后端), `web` (前端), `network` (边缘)。
- **🧪 模块化解耦**: 将前端逻辑拆分为 `ui`, `grid`, `cmd`, `terminal` 等高内聚组件。
