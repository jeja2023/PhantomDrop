## [V0.0.35] - 2026-07-20

### Grok 注册前就绪诊断

- **收信域名自动解析**：Grok 与 OpenAI 注册统一采用“工作流 `account_domain` -> 系统 `account_domain` -> `cloudflare_zone_domain`”的三级回退；空字符串不再遮蔽有效配置，并拒绝 `.local`、IP 地址、带协议或路径的无效域名。
- **可执行就绪报告**：新增受管理会话保护的 `GET /api/workflows/grok/readiness`，集中检查收信域名、`HUB_SECRET`、公网 Hub、Cloudflare Zone、近 30 天邮件记录、代理及 Turnstile 处理能力。
- **失败前置拦截**：硬失败项会在访问 xAI 前阻止任务启动，警告项保留人工决策空间；自动化中心同步展示每项状态、实际域名来源和 Solver 模式，避免任务运行后才暴露基础配置错误。

### 注册链路可靠性

- **动态参数浏览器回退**：HTTP 访问 xAI 注册页或 Next.js 资源被挑战页拦截时，自动使用 Chromium 重新发现 Turnstile Site Key 与 Server Action ID，降低页面结构和边缘风控变化造成的失败率。
- **Turnstile 级联容错**：按 YesCaptcha、本地 Solver、Chromium 顺序尝试；单一处理器失败后继续回退，并保留每一阶段的可诊断错误，不再因首选服务临时故障直接终止。
- **配置校验收口**：工作流保存与启动统一校验公网收信域名、代理和 Solver URL；缺少 `HUB_SECRET`、无可用 Solver/Chromium 等确定性问题会给出明确中文提示。

### 管理界面、开发环境与发布工具

- **Grok 状态面板**：桌面与移动端均可查看注册链路是否就绪、当前域名来源及各检查项结果；保存配置后自动刷新，启动前再次核验并阻止硬失败配置。
- **开发代理修复**：Vite 开发服务器新增 `/auth` 后端代理，恢复本地前后端分离启动时的登录、登出和会话状态请求。
- **发布一致性修复**：版本工具使用当前本地日期生成日志条目，兼容 Windows `CRLF` 的 `Cargo.lock`，并在找不到 Core 包版本时失败退出，避免静默生成版本不一致的发布。
- **文档同步**：更新 README、Grok/Cloudflare 专项说明、验证指南、实施状态和技术审计增量，明确 Cloudflare 域名回退、就绪检查语义及升级边界。

### 版本与升级

- Core、Web、Cloudflare Worker、双 lockfile、侧边栏和 OCI 镜像标签统一升级至 `0.0.35`。
- 本版本不新增数据库迁移。原地升级继续复用现有 `data` 与 `automation` 卷；启用 Grok 邮件闭环时，Hub 和 Worker 必须配置一致的 `HUB_SECRET`。

### 验证

## [V0.0.34] - 2026-07-20

### Grok 自动注册

- **内建注册工作流**：新增 `grok_register_default` / `grok_register` 工作流，自动生成邮箱与密码，并行完成 xAI 邮箱验证、Turnstile、账号创建、SSO 提取和注册后初始化；批量范围为 `1..=50`，并发范围为 `1..=10`，支持运行中协作式取消。
- **动态协议参数发现**：运行时从 xAI 注册页及 Next.js 静态资源发现 Turnstile Site Key 和 Server Action ID，降低页面构建变更导致固定参数失效的风险；认证跳转仅接受受信任的 xAI/Grok 主机。
- **三种 Turnstile 方案**：支持 YesCaptcha、兼容 `/turnstile` 与 `/result` 接口的本地 Solver，以及未配置外部 Solver 时的 Chromium 回退；代理配置同步用于注册会话和验证码任务，带认证代理会在不受支持的 Chromium 路径上明确失败。
- **账号资产落库**：成功账号以 `grok_registered` 状态和 `grok_free` 类型写入现有账号库，Grok `sso` 保存到 `session_token`；单个任务失败不会终止同批其他任务，运行摘要统一报告成功、失败和总数。

### 邮件与验证码可靠性

- **xAI 验证码解析**：新增 xAI/Grok 发件人识别，优先从邮件主题提取 `ABC-123` 格式，再回退纯文本和 HTML 正文，并统一规范化为 `ABC123`。
- **OTP 轮询泛化**：数据库轮询由纯六位数字扩展为六位字母数字验证码，同时保留至少包含一位数字的约束，避免普通单词被误识别；仅查询注册请求发出后的目标收件地址。
- **可诊断超时**：验证码等待按任务超时持续轮询并输出周期状态，超时信息直接提示检查 `account_domain` 与 Cloudflare 邮件路由。

### 管理界面与文档

- **独立 Grok 注册中心**：自动化中心新增与 OpenAI 注册平级的 Grok 一级入口，Grok 配置和操作不再显示在 OpenAI 面板内；独立页面提供批次、并发、代理、YesCaptcha 服务密钥、本地人机验证服务地址和浏览器模式设置，并复用现有工作流保存、启动、终止与实时日志能力。
- **Grok 工作流编排**：工作流设计师展示内建“Grok 自动化注册”卡片，并按 `grok_register` 类型提供批次、收信域名、并发、代理、验证码服务、浏览器模式和超时参数编辑；支持一键复制为可独立修改、调度和删除的自定义 Grok 工作流。
- **登录密码可见性**：管理员登录密码框新增眼睛图标，可在隐藏与明文显示之间切换；登录成功或重新打开登录框时自动恢复隐藏，密码仍只保存在当前表单状态中。
- **专项运行手册**：README 补充快速启用步骤，新增 `docs/Grok自动注册说明.md`，记录配置边界、Solver 接口契约、凭据落库位置、安全注意事项与故障排查。
- **版本一致性**：Core、Web、Cloudflare Worker、双 lockfile、侧边栏、OCI 镜像标签和验证命令统一升级至 `0.0.34`。

## [V0.0.32] - 2026-07-19

### 管理认证与机器通信彻底分离

- **用户名/密码登录**：管理端改为独立的 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 首次初始化模型，登录表单、错误反馈、浏览器自动填充语义和系统设置凭据管理同步完成。
- **Argon2id 密码存储**：管理员密码使用独立随机盐进行 Argon2id 哈希，数据库仅保存用户名和密码哈希，不保存原始密码。用户名或密码更新后旧会话立即失效。
- **Cookie 会话收敛**：管理 API、SSE、内建控制台与调试资源仅接受 `HttpOnly`、`SameSite=Strict` 管理会话 Cookie；移除管理面的 Bearer、`X-Auth-Token`、原始共享密钥认证和前端本地密钥存储。
- **登录防爆破**：保留按客户端 IP 的失败次数窗口限制，统一使用常量时间比较，并为登录、修改凭据和未初始化状态提供明确错误响应。
- **旧密钥安全迁移**：升级后删除遗留 `auth_secret`，且不把旧机器密钥自动迁移为管理员密码，避免管理权限与机器权限继续共用同一秘密。

### HUB_SECRET 边界重构

- **只用于机器间通信**：`HUB_SECRET` 现在只验证 Cloudflare Worker -> Hub `/ingest` 的 `X-Hub-Secret`，不能登录管理端，也不能访问任意 `/api/*`、`/stream`、`/console` 或 `/debug/*`。
- **核心运行可选**：不使用邮件 Worker 时可不配置 `HUB_SECRET`，系统和管理控制台照常启动，`/ingest` 以 `503` 明确关闭；启用 Worker 时 Hub 与 Cloudflare Secret Store 必须使用同一强随机值。
- **取消明文缓存**：Cloudflare 自动化不再从管理 API读取或向 `.automation/cloudflare-config.json` 写入 `HUB_SECRET`；旧缓存字段在下次初始化或部署时自动删除。
- **自动化链修复**：后端自动化管理器内部读取已保存的 Cloudflare 非登录配置并注入脚本，脚本无需匿名调用受保护的设置 API；部署完成后由后端保存公网 URL 与隧道状态。

### 安全加固

- **SSRF 防护升级**：Webhook、CPA、Sub2API 等出站地址执行协议、主机、解析结果和内网地址校验，并将已校验 DNS 结果固定到请求客户端，降低 DNS rebinding 风险。
- **敏感信息最小化**：设置 API 对 Cloudflare、CPA、Sub2API 和 OAuth 凭据统一脱敏；账号列表返回最小化 DTO，完整令牌仅在明确的单账号详情路径提供。
- **网关访问控制**：OpenAI 兼容网关支持密钥到账号池映射、请求体上限、速率限制、失败账号切换和令牌刷新，生产模式拒绝空密钥配置。
- **浏览器与调试隔离**：默认关闭生产调试资产，浏览器启动参数按环境收敛，敏感认证文件写入改为显式开关。
- **CORS 收口**：生产来源使用明确白名单与凭据模式，移除废弃的 `X-Auth-Token` 和跨域 `X-Hub-Secret` 管理头。

### 数据一致性与可靠性

- **事务型 Webhook Outbox**：邮件写入与 Webhook 投递任务在同一事务创建，后台工作器按退避策略重试，避免进程重启或回调故障造成事件丢失。
- **原子账号路由**：账号选取、占用与使用时间更新收敛为数据库原子操作，降低并发网关请求重复命中同一账号的风险。
- **数据库索引与完整性迁移**：新增 `202607190001_performance_and_integrity.sql`，覆盖邮件分页、OTP 轮询、工作流历史、账号池和 Outbox 高频路径。
- **SSE 丢包恢复**：广播接收器落后时发送 `resync_required` 事件，前端触发状态重拉，不再静默丢失实时事件。
- **工作流生命周期**：加入并发信号量、启动恢复、中断状态持久化和优雅关闭，限制工作流、代理检测与外部上传的资源峰值。

### 前端与工程质量

- **按视图懒加载**：收件中心、自动化中心和系统设置拆分为异步块，减少首屏主包与非当前页面的初始化开销。
- **认证体验完整化**：登录弹窗改为用户名和密码双字段，显示服务端错误与提交状态；设置页提供需校验当前密码的用户名/密码修改流程。


- **依赖与类型收敛**：移除未使用前端依赖与旧设置字段，完成 React、TypeScript、Vite、Worker 类型和 Rust SQLite 专用依赖链整理。
- **CI 与容器固化**：锁定 Node、Rust 和 cloudflared 版本及校验和；容器使用非 root 用户、健康检查、日志轮换和持久化卷；CI 覆盖格式、Clippy、测试、构建、类型、漏洞扫描与 Docker 构建。
- **验证基线**：Rust 测试、严格 Clippy、前端 lint/生产构建、Worker typecheck、npm audit、cargo audit、PowerShell AST 和 HTTP 认证隔离均纳入 `VERIFY.md`。

### 版本与升级说明

- Core、Web、Cloudflare Worker、lockfile、侧边栏和 OCI 镜像标签统一升级至 `0.0.32`。
- 从 `V0.0.31` 或更早版本升级时，首次启动 `V0.0.32` 必须提供至少 12 个字符的 `ADMIN_PASSWORD`；用户名默认 `admin`，可通过 `ADMIN_USERNAME` 覆盖。

- 已初始化数据库后，环境中的 `ADMIN_PASSWORD` 不会覆盖数据库凭据；后续通过系统设置修改。未启用 Worker 时可保持 `HUB_SECRET` 为空。

## [V0.0.31] - 2026-05-24

### 🛸 OAuth 联合提纯全面升级为“混合模式”免人工闭环 (Mixed OAuth Automation & Pure)
- **仿真浏览器自动调起与导航**：

- **验证人机后彻底零干预录入**：

- **零复制、零粘回双轨闭环**：
  整个联合注册提纯过程完美摆脱了所有繁琐的外部复制链接、新标签页打开以及手工复制回调 URL 粘回的物理步骤，用户体验实现革命性飞跃。同时保留了原有的手工复制粘贴作为极其稳健的兜底方案。
