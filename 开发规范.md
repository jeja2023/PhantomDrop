# 开发规范

为了保证项目的质量与后续的可维护性，所有参与本项目的开发者必须遵循以下规范：

## 1. 注释与提示语言
- **中文注释**：代码中的所有注释必须使用**简体中文**编写，解释逻辑、参数含义及复杂算法。
- **中文提示**：用户交互界面、命令行输出、日志记录等所有面向用户的提示信息必须使用**简体中文**。

## 2. 编码风格
- **禁止调试信息**：代码提交前必须清理所有调试用的 print 或 console.log 等冗余输出。
- **项目日志**：应用运行时的日志输出必须使用中文，确保日志信息清晰直观。

## 3. 目录与命名
- **目录结构**：项目目录应根据功能分门别类，层次分明。
- **文件命名**：文件名应当简短且具有描述性，禁止出现 generic_file1 这种无意义命名。
- **禁止冗余后缀**：URL、路由等命名中禁止出现 `backend`、`frontend`、`html` 等冗余前后缀。

## 4. 文档规范
- **中文命名**：相关的技术方案、使用手册、说明文档等命名尽量使用中文。

## 5. 数据库与发布
- **开发数据源**：日常开发固定使用持久化的 `core/phantom_core.db`；`.codex-tmp` 和 `artifacts` 下的数据库只用于隔离测试，不得作为开发或生产数据源。
- **管理员初始化**：`ADMIN_USERNAME` / `ADMIN_PASSWORD` 只用于空数据库首次初始化，禁止把真实管理员密码写入代码、文档、日志或版本库；后续改密通过“账户与登录”完成。
- **生产升级**：升级前停止写入并备份现有 `data` 卷，升级时必须复用 `data` 与 `automation` 命名卷，禁止以新空卷替换旧数据。
- **版本同步**：发布版本通过 `tools/release-tool` 统一更新 Core、Web、Worker、lockfile、侧边栏和 OCI 标签；每个版本必须同步维护 `更新日志.md`、`README.md` 与 `VERIFY.md`。
