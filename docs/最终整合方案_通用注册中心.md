# PhantomDrop: 通用“网站注册”集成方案 (基于 OpenAI 试点)

本方案在保持 Rust 核心高性能注册引擎的基础上，重点优化了前端的架构灵活性，使其支持多平台注册能力的水平扩展。

---

## 1. 架构调整 (Generalized Architecture)

### 1.1 前端：分层标签系统
- **主导航**: 在系统左侧 Sidebar 增加“网站注册” (`Registration`) 菜单。
- **视图层**: 创建 `RegistrationView.tsx`，内置二级 Tab 导航（目前包含：OpenAI）。
- **组件复用**: 提取通用的 `RegistrationConfig` 和 `WorkflowMonitor` 组件，使未来新增平台仅需定义配置字段而无需重写 UI 逻辑。

### 1.2 后端：工作流分类
- **扩展 Kind**: `WorkflowKind` 新增 `OpenAIRegister`。
- **目录结构**: 
  - `core/src/register/`: 存放通用的注册辅助函数（如 DB 操作、统一分发逻辑）。
  - `core/src/openai/`: 存放 OpenAI 专属的协议套件（PoW, OAuth, Sentinel）。
